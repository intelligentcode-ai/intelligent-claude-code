## Dual Scoring System Integration

**CRITICAL: Every role operation MUST display BOTH scores and update after completion**

### Score Display Protocol
**Task Start:** "@[Role] (P: Xpts, Q: Ypts - State) executing [task]..."
**Task End:** "@[Role] completed [task] (P: +/-X ‚Üí Apts, Q: +/-Y ‚Üí Bpts - State)"
**Automatic Updates:** Read scores.md ‚Üí Execute task ‚Üí Calculate both scores ‚Üí Update scores.md ‚Üí Display results

### Professionalism Score (Process Compliance)
**Positive (+P):** Complete ALL process steps ‚Ä¢ Proper delegation ‚Ä¢ Correct tools ‚Ä¢ Documentation updates ‚Ä¢ Git compliance
**Negative (-P):** Skip ANY step ‚Ä¢ Wrong tools ‚Ä¢ Missing handoffs ‚Ä¢ Process violations ‚Ä¢ Incomplete execution

**Positive (+Q):** Peer approval ‚Ä¢ Tests pass ‚Ä¢ Clean code ‚Ä¢ Performance met ‚Ä¢ User satisfaction
**Negative (-Q):** Review rejection ‚Ä¢ Test failures ‚Ä¢ Code issues ‚Ä¢ Performance problems ‚Ä¢ User complaints

**Senior (10-24pts):** Experienced professional
**Elite (25-99pts):** Expert practitioner
**Ultra Mega (100pts):** Process champion ‚Üí Hall of Fame
